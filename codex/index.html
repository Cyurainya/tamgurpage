三步完成配置

每一步一张大图，照着截图操作即可。

模型中心编辑配置应用管理
1

添加模型

进入模型中心，点击“添加模型”，选择 OpenAI / Anthropic API 类型。

截图 1：在模型中心点击添加模型。
2

填写 API 配置

按下面的表单提示填写 Tamgur API 信息。

表单填写 Tips
  1. Name：随意填写，用于区分模型配置。
  2. OpenAI URL：https://api.tamgur.tech/v1
  3. Anthropic URL：留空。
  4. Model ID：填写需要使用的模型，直接从模型广场复制。
  5. API Key：填写你的 API Key，请记得选择正确的分组。
截图 2：填写地址、模型 ID 和 API Key。
3

绑定 Codex

进入应用管理，选择 Codex Desktop，绑定刚添加的模型，然后点击“启动应用”。

截图 3：为 Codex Desktop 选择模型并启动。

配置要点

填错最多的地方，按这里核对。

填写规则

  • 名称只是显示名，可以写成 GPT5.5、KK-GPT、Codex 中转站等。
  • OpenAI 地址填写 https://api.tamgur.tech/v1。
  • 模型 ID 要和中转站后台支持的模型名称一致，建议直接复制。
  • API 密钥填中转站令牌 Key，如果有分组，记得选对分组权限。

配置模板

把下面内容替换成你自己的中转站信息即可。

名称：GPT5.5
OpenAI 地址：https://api.tamgur.tech/v1
Anthropic 地址：留空
模型 ID：请从模型广场复制
API 密钥：sk-xxxxxxxxxxxxxxxx
如果 Codex 没有立即切换，重新点击“启动应用”即可。使用时保持 EchoBird 后台运行。
